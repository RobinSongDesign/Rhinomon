# Rhinomon — Rhino 视口桌宠插件 PRD

**版本**: v1.0（已确认）
**日期**: 2026-07-05
**状态**: 已通过 grill 评审，待开发

---

## 1. 概述

Rhinomon 是一个运行在 Rhino 8 内的桌宠插件。宠物以像素风 2D 精灵的形式生活在建模视口中，**完全通过显示管线（DisplayConduit）绘制，不向文档写入任何几何数据**。它对用户的点击和建模操作做出反应，空闲时会在视口里漫步、爬上用户创建的几何体、睡觉。

**核心信条：宠物是陪伴，不是负担。任何情况下不得影响建模主流程。**

---

## 2. 目标与非目标

### 目标
- 一只低调可爱的像素桌宠，让建模过程更有陪伴感
- 性能开销低到在重模型上可以完全忽略（见 §7 硬指标）
- 达到 food4Rhino 公开发布质量：Yak 打包、英文文档、永不崩溃 Rhino

### 非目标（v1 明确不做）
| 项 | 说明 |
|---|---|
| 声音 | 无任何音效 |
| Mac 支持 | 架构不排除，但不开发不测试 |
| Grasshopper 联动 | 不做 |
| 多只宠物同场 | 全局唯一一只 |
| 文字对话气泡 | 反应全部用像素表情图标，天然免多语言 |
| 遥测/统计 | 不收集任何数据 |
| 养成/好感度持久化 | 仅会话内心情值，重启归零（v2 再评估） |
| Eto 设置面板/工具栏 | v1 仅命令行（v2 再评估） |

## 4. 功能需求

### 4.1 渲染（F1）
- 单例 `DisplayConduit`，在 `DrawForeground` 阶段以屏幕空间绘制宠物 sprite
- 精灵表以 `DisplayBitmap` 加载（GPU 缓存），用 `DrawSprite` / 2D 绘制 API 输出
- 逻辑分辨率 32×32，默认 2× 整数缩放（64px），可调 1×/2×/3×，DPI 感知
- 仅在当前激活视口绘制；conduit 对非激活视口的回调立即 return
- 表情反应用宠物头顶的像素表情图标（❤ ❗ ❓ 💤 ✨ 等），同为 sprite，无文字

### 4.2 动画引擎（F2）
- 空闲动画由定时器驱动，目标 4–6 fps，仅 invalidate 激活视口
- **避让规则（最高优先级）**：以下任一状态成立时定时器立即暂停，只在用户自己触发的重绘里"搭便车"绘制当前帧：
  - 有命令正在运行（`Command.BeginCommand`→`EndCommand` 之间）
  - 视图正在动态操作（旋转/平移/缩放拖拽中）
  - 鼠标左键拖拽中
- 帧时间采样驱动自动降级（见 §7）

### 4.3 交互：点击（F3）
- `MouseCallback` 监听左键按下
- 拦截条件（同时满足）：① 无命令运行；② 点击位置在宠物当前屏幕包围盒内
- 拦截后触发"被摸"反应（开心动画 + ❤ 表情），事件不传给 Rhino
- 命令运行期间 `MouseCallback` 直接透传，不做任何判断逻辑

### 4.4 交互：建模反应（F4）
- 订阅 `RhinoDoc.AddRhinoObject` / `DeleteRhinoObject` / `ReplaceRhinoObject` / `UndoRedo`、`Command.BeginCommand/EndCommand`
- 事件处理函数内部**只做整型计数**，零分配、不读几何
- `EndCommand` 时按计数合并为一次反应：
  - 新建了对象 → 开心/✨
  - 大批量删除（≥10）→ 惊讶/❗
  - Undo → 疑惑/❓
- 全局反应冷却 8 秒；冷却中的触发直接丢弃

### 4.5 空闲行为（F5）
空闲 = 无命令、无鼠标移动（视口内）、无视图变化、无文档变化。

| 空闲时长（活泼档默认） | 行为 |
|---|---|
| 10s | 在视口底边漫步 |
| 30s | 扫描可见对象 → 选目标 → 走过去"爬上"包围盒顶部 |
| 2min | 在落脚点睡着（1 fps 呼吸动画或静帧） |

- 档位：活泼(10s/30s/2min)（默认）、默认(30s/90s/5min)、佛系(2min/10min/30min)
- **打断规则**：
  - 仅视图旋转/平移 → 宠物留在世界锚点上，每帧一次点投影跟随
  - 命令开始/文档编辑 → 立即苏醒，跳下，回到"家"位置（视口右下角）
  - 锚点对象被删除或锚点移出屏幕 → 宠物坠落小动画，回家

### 4.6 爬几何算法（F6）
1. 空闲触发时遍历文档可见对象，读取**缓存的** `BoundingBox`（不触发重算）
2. 将 bbox 角点投影到屏幕，筛选屏幕内、尺寸合适（投影高度 ≥ 宠物身高）的候选
3. 随机/就近选一个，落脚点 = 屏幕空间 bbox 顶边中点对应的世界坐标点（取 bbox 顶面中心）
4. 宠物沿视口底边走到目标横坐标 → 播放"爬"动画上升到落脚点
5. 已知妥协：非盒状物体（球/斜面）宠物会轻微悬浮于轮廓上方——v1 接受
6. 扫描频率上限：每次空闲episode最多一次；对象数 >5000 时跳过爬高行为（直接漫步/睡觉）

### 4.7 心情系统（F7，会话内）
- 简单枚举状态机：平静 / 开心（被摸、建模反应后一段时间）/ 无聊（长时间无互动，发呆动画）
- 不持久化，重启 Rhino 归零

### 4.8 命令（F8）
单命令 `Rhinomon`：
- 直接回车 → 开关宠物（Toggle）
- 选项：
  - `Pet` = Clawd / Crab / Nova
  - `Scale` = 1 / 2 / 3
  - `Activity` = Lively / Normal / Chill
  - `Hide` = 永久隐藏（kill switch，含确认；再次运行 `Rhinomon` 恢复）
- 所有设置经 `PlugIn.Settings` 持久化，重启 Rhino 保留

## 6. 架构草图

```
RhinomonPlugin (PlugIn, 延迟加载 LoadTime=WhenNeeded→由命令触发)
├── PetConduit        : DisplayConduit，只画不想
├── PetEngine         : 状态机 + 动画帧调度 + 空闲计时（单例，UI 线程）
├── ActivityMonitor   : 命令/鼠标/视图/文档事件 → 空闲判定 + O(1) 反应计数
├── ClickInterceptor  : MouseCallback，条件式拦截
├── PerchScanner      : 空闲时 bbox 扫描 + 投影选点
├── SpriteAtlas       : DisplayBitmap 加载与缓存
├── PerfGovernor      : 帧时采样 → 降级/休眠决策
└── RhinomonCommand   : 命令 + 设置持久化
```

- 全部逻辑跑在 Rhino UI 线程（事件与绘制天然同线程，无锁）；任何单次工作 <1ms，超过的（如 bbox 扫描）分摊或设上限
- **conduit 与所有事件回调外层 try/catch**；同一异常连续出现 3 次 → 插件自动禁用宠物并在命令行提示（绝不让异常泄漏进渲染管线导致 Rhino 崩溃）

## 7. 性能硬指标（验收标准）

| # | 指标 | 数值 |
|---|---|---|
| P1 | conduit 单帧自身绘制耗时 | < 0.3 ms |
| P2 | 插件总内存增量 | < 30 MB |
| P3 | 场景重绘耗时 > 25 ms 时 | 动画自动降到 1–2 fps |
| P4 | 场景重绘耗时 > 60 ms 时 | 定时器完全停止，宠物变纯事件驱动静态贴图 |
| P5 | Rhino 启动时间影响 | 0（延迟加载） |
| P6 | 命令运行期间宠物产生的额外重绘次数 | 0 |
| P7 | 事件回调单次执行 | O(1)、零堆分配 |
| P8 | kill switch | `Rhinomon Hide` 后所有定时器/事件/conduit 完全卸载 |

验收方法：构造 10 万面片测试模型，用 Rhino `TestMaxSpeed` 对比开关宠物前后的视口帧率，差异须 <2%。

## 9. 风险与缓解

| 风险 | 缓解 |
|---|---|
| 定时重绘在重场景被感知 | P3/P4 两级自动降级 + TestMaxSpeed 验收 |
| MouseCallback 干扰第三方插件的鼠标钩子 | 命令期间完全透传；只拦截极小区域；充分测试常见插件共存 |
| bbox 扫描在超大文档卡顿 | 对象数 >5000 直接跳过爬高；扫描每 episode 一次 |
| conduit 异常导致 Rhino 崩溃 | 全回调 try/catch + 3 次异常自动禁用 |
| Rhino 8 各 service release 显示管线差异 | 只用稳定公开 API（DrawSprite/DisplayBitmap），CI 对最新 SR 冒烟 |

## 12. v2 范围（2026-07-05 确认）

v1 真机验证通过后追加。已确认决策：**跳过原 v2 备选的 2D ZBuffer 轮廓行走**——其能力被
2.5D 世界模式完全覆盖，避免同一功能实现两次。

### 12.1 控制面板（F9）
- Eto 停靠面板，命令 `Rhinomon Panel` 打开
- 控件：宠物选择（内置 3 只 + 自定义）、Scale、Activity、显示模式（2D/2.5D）、
  世界尺寸（2.5D 用，模型单位）、开关宠物、Hide（kill switch，带确认）
- 所有改动即时生效并持久化；与命令行选项等价，互为镜像

### 12.2 自定义宠物导入（F10）
- 用户可导入符合 CONTRACT.md 精灵表规格的 PNG，或导入单张 PNG/image2 输出图；
  导入端会归一化为内部 256×224 精灵表，不把非契约图片交给运行时 atlas
- 副本存入 `%AppData%\Rhinomon\pets\`，与内置宠物并列出现在选择列表
- 表情图标仍用内置；自定义表情不在 v2 范围

### 12.3 伪透视 2.5D 世界模式（F11）— 已实现（spike 真机验证通过）
- 模式开关：Screen 2D（默认，v1 行为）/ World 2.5D
- 渲染：世界空间 sprite（`DrawSprite` 世界重载，绝对世界尺寸，随镜头远近缩放），
  绘制在深度测试通道（PostDrawObjects）→ 被几何体自然遮挡；帧预先 4× 最近邻放大
  抑制滤波糊化
- 运动：世界坐标导航，**拓扑仅用包围盒面**（bbox 顶面=地面、侧面=墙、世界 XY 面=地板），
  纯显示侧运动学，禁止渲染网格射线求交——性能红线延续 v1
- 行为映射：地面漫步 → 走到目标 bbox 底边 → 沿垂直面爬升 → 顶面漫步/睡觉；
  锚定对象被删 → 重力坠落回地板
- 表情图标在 2.5D 下仍画在屏幕空间（投影点上方）：宠物被完全遮挡时表情仍可见，
  兼作"宠物在哪"指示
- 点击命中：世界位置投影回屏幕做矩形测试，ClickInterceptor 逻辑不变
- 性能：所有 v1 硬指标（§7）不变；conduit 每帧新增成本仅一次点投影
- 真机验证：2026-07-05 spike 已确认 sprite 在 PostDrawObjects 通道深度测试生效，
  DrawSprite 世界重载随相机距离正确缩放，4× 预放大像素清晰度可接受。
